数据库（一）

Posted on 2020-05-20 In DB

数据库

1.数据库

    将大量数据保存起来，通过计算机加工而成的可以进行高效访问的数据集合称为数据库（Database，DB）。用来管理数据库的计算机系统称为数据库管理系统（Database Management System，DBMS）。

2.数据库管理系统（DBMS）的种类

2.1 层次数据库（Hierarchical，HDB）

    层次结构的方式表达数据，在关系型数据库出现后，现在很少使用。

2.2 关系数据库（Relational Database，RDB）

    关系型数据库是现阶段使用最广泛的数据库，与Excel一样采用行和列组成的二维表来管理数据库，同时还使用专门的SQL（Structured Query Language，结构化查询语言）对数据库进行操作。
    这类DBMS称为关系型数据库管理系统（Relational Database Management System，RDBMS）。比较具有代表性的RDBMS有以下五种：

Oracle Database: 甲骨文公司的RDBMS
SQL ServerL: 微软公司的RDBMS
DB2: IBM公司的RDBMS
PostgreSQL: 开源的RDBMS
MySQL: 开源的RDBMS

2.3 面向对象数据库（Object Oriented Database，OODB）

    把数据已经对数据的操作集合起来以对象为单位进行管理，因此得名。面向对象数据库就是用来保存这些对象的数据库。

2.4 XML数据库（XML Database，XMLDB）

    XML数据库可以对XML形式的数据进行高速处理。

2.5 键值存储系统（Key-Value Store，KVS）

    是一种单纯用来保存查询所使用的主键（Key）和键值（Value）组合的数据库。键值存储系统现用于对大量数据进行高速查询的web服务中。

3.表的结构

    用来管理数据的二维表在关系数据库中简称为表。根据SQL语句的内容返回的数据同样必须是二维表的形式，返回结果不是二维表的SQL语句则无法执行。

4. SQL概要

4.1 SQL语句及其种类

    SQL用关键字，表名，列名等组合而成的一条语句（SQL语句）来描述操作的内容。关键字是指那些含义或使用方法已事先定义好的英语单词，存在包含”对表进行查询”或者”参考这个表”等各种意义的关键字。
    根据对RDBMS赋予的指令种类的不同，SQL语句可以分为以下三类：

  • DLL
    DDL（Data Defination Language，数据定义语言）用来创建或删除存储数据用的数据库以及数据库中的表等对象。DDL包含以下几种指令。

CRETAE: 创建数据库和表等对象
DROP: 删除数据库和表等对象
ALTER: 修改数据库和表等对象的结构

  • DML
    DML（Data Manipulation Language，数据操纵语言）用来查询或者变更表中记录。DML包含以下几种指令。

SELECT: 查询表中的数据
INSERT: 向表中插入新数据
UPDATE: 更新表中的数据
DELETE: 删除表中的数据

  • DCL
    DCL（Data Control Language，数据控制语言）用来确认或者取消对数据库中的数据进行的变更。DCL包含以下几种指令。

COMMIT: 确认对数据库中数据进行的变更
ROLLBACK: 取消对数据库中数据进行的变更
GRANT: 赋予用户操作权限
REVOKE: 取消用户的操作权限

    使用最多的是DML。

4.2 SQL的基本书写规则

  • SQL语句要以分号(;)结尾

        一条SQL语句可以描述一个数据库操作。在RDBMS当中，SQL语句也是逐条执行的。

  • SQL语句不区分大小写

        SQL不区分关键字大小写。例如，不管写成SELECT还是select，解释都是一样的。表名和列名也是如此。但建议定义一个规则：关键字大写，表名的首字母大写，其余（列名）小写。
        但需要注意的是，插入到表中的数据是区分大小写的。

  • 常数的书写方式是固定的

        SQL语句常常需要直接书写字符串、日期或者数字。SQL语句含有字符串时，用单引号(‘ ‘)将字符串括起来说明是一个字符串（如’abc’）；日期同样用单引号(‘ ‘)将日期括起来（如(‘2020-5-20’)），数字书写不需要任何标识，直接写数字即可（如 1000）。

  • 单词需要用半角空格或者换行来分隔

        SQL语句的单词之间需要用半角空格或者换行来分隔。示例如下：

5.3 命名规则

    我们只能使用半角英文字母、数字、下划线(_)作为数据库、表和列的名称。且名称必须以半角英文字母开头。在同一个数据库中不能创建名称相同的表，也不能在同一个表中创建名称相同的列。

5.4 数据类型的指定

    所有列都必须指定数据类型。数据类型表示数据的种类，包括数字型、字符型、日期型等。
    数据类型主要有四种：

5.4.1 INTEGER型

    用来指定存储整数的列的数据类型（数字型），不能存储小数。

5.4.2 CHAR型

    CHAR是CHARACTER（字符）的缩写，用来指定存储字符串的列的数据类型（字符型）。可以像CHAT(10)或者CHAR(200)这样，在括号中指定该列可以存储的字符串的长度（最大长度），当输入字符数超过最大长度时无法输入到该列中。
    字符串以定长字符串的形式存储在被指定为CHAR型的列中。如用CHAR(8)存储'abc'，则实际存储的是'abc_ _ _ _ _'。且在列中，’abc’与’ABC’是不同意义的字符串。

5.4.3 VARCHAR型

    同CHAR一样用来存储字符串列的数据类型（字符串类型），也可以通过在括号内控制长度。该类型的列是可变长字符串的形式来保存字符串的。如果输入字符未达到定长时，VARCHAR不会自动补充空格，即VARCHAR(8)输入一列’abc’，则存储时也为’abc’，不会像CHAR一样自动填充空格。

5.4.4 DATE型

    用来指定存储日期（年月日）的列的数据类型（日期型）。

5.5 约束的设置

    约束是指除了数据类型约束以外的对存储数据进行限制或追加条件的功能。之前的表中定义了两个约束，NOT NULL表示该列为必填项，不能为空；此外最后一行还有PRIMARY KEY (product_id)，这是用来给product_id设置主键约束的。所谓键，就是指在特定数据时使用的列的组合。将product_id设置为主键意味着可以通过该列取出特定的商品数据了，所以主键的数据一定唯一，否则无法读取特定的一行。
